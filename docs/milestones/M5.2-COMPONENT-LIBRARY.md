## Tasks

### Phase A: Replace Active Components (breaking changes — update all call sites)

#### A1. Button

**Blast radius:** 4 files, ~10 instances

**API migration:**

| Current | shadcn | Notes |
|---------|--------|-------|
| `variant="primary"` | `variant="default"` | shadcn's default = our primary |
| `variant="secondary"` | `variant="outline"` | Our secondary style maps to shadcn outline |
| `variant="ghost"` | `variant="ghost"` | Same name |
| `variant="destructive"` | `variant="destructive"` | Same name |
| `size="sm"` | `size="sm"` | Same name |
| `size="md"` | `size="default"` | shadcn uses "default" not "md" |
| `size="lg"` | `size="lg"` | Same name |

**Steps:**
1. Generate shadcn button: `pnpm dlx shadcn@latest add button`
2. Customize variants to match our token system (navy-900 primary, surface-1 secondary, etc.)
3. Update all 4 consumer files: rename `variant="primary"` → `variant="default"`, `size="md"` → `size="default"`
4. Update barrel export in `index.ts`
5. Verify all button usages render correctly

#### A2. Dialog

**Blast radius:** 3 files, ~15 instances

**API migration:**

| Current Export | shadcn Equivalent | Notes |
|---|---|---|
| `Dialog` | `Dialog` | Same — Radix Root |
| `DialogTrigger` | `DialogTrigger` | Same |
| `DialogClose` | `DialogClose` | Same |
| `DialogContent` | `DialogContent` | Loses custom `size` prop — add back via `cn()` + className |
| `DialogHeader` | `DialogHeader` | Same pattern |
| `DialogTitle` | `DialogTitle` | Same |
| `DialogDescription` | `DialogDescription` | Same |
| `DialogBody` | _(none)_ | Keep as custom extension — simple `<div>` wrapper |
| `DialogFooter` | `DialogFooter` | Same pattern |

**Key concern:** Our `DialogContent` accepts `size="sm|md|lg|xl"` which controls `max-width`. shadcn doesn't have this. Options:
- **Option A:** Pass size via className at each call site (`className="max-w-lg"`)
- **Option B:** Extend shadcn's `DialogContent` with our `size` prop (recommended — minimal diff)

**Steps:**
1. Generate shadcn dialog: `pnpm dlx shadcn@latest add dialog`
2. Add `size` prop extension to `DialogContent`
3. Add `DialogBody` export (shadcn doesn't include this)
4. Update barrel export — ensure all current export names still exist
5. Update 3 consumer files if any API changed
6. Verify all modals open/close correctly

## Risks

| Risk | Mitigation |
|------|-----------|
| shadcn button API differs from ours | Only 4 files use Button — small migration surface. Map variant names explicitly. |
| shadcn Dialog missing our `size` prop | Extend `DialogContent` with size prop — shadcn components are meant to be customized. |
| shadcn generates components that reference default gray tokens | Our CSS variable aliases already map shadcn tokens to our palette. Generated components work out of the box. |
| Adding Sonner changes toast behavior | Toast module is currently unused (0 call sites) — no behavior change. |
| New components bloat bundle | shadcn components are copy-pasted source, not a package dependency. Tree-shaking handles unused exports. |
