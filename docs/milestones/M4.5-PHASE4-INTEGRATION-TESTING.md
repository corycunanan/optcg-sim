# M4.5 Phase 4 — Integration Testing

## Agent Prompt

You are picking up M4.5 at Phase 4: Integration Testing. Phases 1-3 are complete — all card effect schemas are encoded, all deferred patterns are resolved, and TypeScript compiles cleanly. Your job is to write integration tests that verify the engine produces correct game states.

### Context

Read these files first to understand the project:
- `CLAUDE.md` — project conventions, codebase map
- `docs/milestones/M4.5-SCHEMA-QA.md` — full M4.5 status, what was built in Phase 3
- `docs/game-engine/DEFERRED-CARD-EFFECTS.md` — all deferred patterns and their resolutions
- `workers/game/src/__tests__/` — existing test files (may be incomplete/stale)

### Test Infrastructure

The game engine lives in `workers/game/src/engine/`. Tests go in `workers/game/src/__tests__/`. The test runner is Vitest (check `workers/game/package.json` for the test script). Read existing test files to understand the test helpers, how GameState is constructed, and how `cardDb` is mocked.

Key engine entry points to test:
- `execute.ts` — `executeAction()` dispatches PLAY_CARD, ATTACH_DON, etc.
- `validation.ts` — `validate()` checks action legality
- `modifiers.ts` — `getEffectiveCost()`, `getEffectivePower()`, hand-zone modifiers
- `conditions.ts` — `evaluateCondition()` for all condition types
- `pipeline.ts` — full event pipeline (action → execute → trigger → resolve)
- `effect-resolver/` — individual action handlers + resume handlers

### What to Test

Write at least 20 tests covering these critical patterns. Each test should construct minimal game state, execute an action or evaluate a function, and assert the resulting state.

#### 1. Hand-Zone Cost Modifiers (5 tests)
- Card in hand with `zone: "HAND"` permanent block gets cost reduced when condition is met
- Card in hand does NOT get cost reduced when condition fails
- Field-to-hand modifier (OP01-067 pattern): Crocodile on field reduces blue Event costs in hand
- `validatePlayCard` accounts for hand-zone cost reductions (player can play card they couldn't otherwise afford)
- Multiple hand-zone modifiers stack correctly

#### 2. REVEAL_CONDITIONAL (3 tests)
- Reveal top card matching filter → conditional action fires
- Reveal top card NOT matching filter → conditional action skipped
- REVEALED_CARD_PROPERTY condition evaluates filter correctly

#### 3. SEARCH_TRASH_THE_REST Resume (2 tests)
- Player selects a card → card goes to hand, rest go to trash
- Player selects a card with `rest_destination: "BOTTOM"` → rest go to deck bottom

#### 4. One-Time Modifiers / NEXT_EVENT_COST_REDUCTION (3 tests)
- APPLY_ONE_TIME_MODIFIER creates a RuntimeOneTimeModifier in state
- `getEffectiveCost` applies unconsumed one-time modifier matching the card
- `consumeOneTimeModifiers` marks the modifier as consumed after play
- `matchesOneTimeFilter` respects name, costMin, traits, card_type filters

#### 5. TURN_LIFE_FACE Costs (2 tests)
- Effect with `TURN_LIFE_FACE_UP` cost: cost handler flips life card, effect fires
- Effect with `TURN_LIFE_FACE_DOWN` cost: fails if no face-up life cards exist

#### 6. Dynamic Values (2 tests)
- `LEADER_BASE_POWER` resolves to the leader's printed power
- `SET_BASE_POWER` with dynamic GAME_STATE value applies correctly

#### 7. Stage Support in SEARCH_AND_PLAY (1 test)
- Resume handler plays a Stage card (not just CHARACTER)

#### 8. Condition Evaluator Edge Cases (2 tests)
- `DON_GIVEN` with `SPECIFIC_CARD` mode checks source card's attached DON
- `COMPARATIVE` with negative margin (DON field count at least 2 less)

### Acceptance Criteria

- At least 20 tests pass
- Tests cover all 7 deferred patterns that were implemented
- No test relies on external services or real card data — use mock CardData
- Tests are deterministic (no randomness unless seeded)
- `npx vitest run` passes in `workers/game/`

### Do NOT

- Modify any engine code — tests only
- Skip patterns because they "seem simple" — test them anyway
- Write tests that just check TypeScript types — test runtime behavior
- Add new dependencies without checking `package.json` first
