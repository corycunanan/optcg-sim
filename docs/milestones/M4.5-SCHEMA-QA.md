## Context

M4 delivered a complete first pass: every released OPTCG card has an effect schema encoded across 50 schema files. This was done at speed by encoding agents, which means systematic errors are expected â€” wrong enum values, missing back-references, duration mismatches, action types used without engine handlers, and cards that should have been deferred but weren't.

M4.5 is the QA gate between "all cards encoded" and "all cards encoded correctly." The goal is to build confidence in the schema corpus before using it for gameplay.

### Guiding Principle: Err on the Side of Caution

**A wrong schema is worse than a missing one.** A deferred card falls back to `MANUAL_EFFECT` and is visibly incomplete. A wrong schema silently produces incorrect gameplay â€” the wrong targets get selected, the wrong conditions gate an effect, or an action resolves with the wrong semantics. Players won't know the card is broken; they'll just lose games to bugs.

During the first encoding pass, there were cases where an encoding agent didn't know how to handle a pattern and **substituted a more familiar one instead of deferring**. For example, encoding a reveal-conditional as an unconditional draw, or mapping a hand-zone cost reduction to a field modifier. These "low-confidence encodings" are the most dangerous class of error because they pass structural validation â€” the schema is well-formed, it just doesn't match the card.

**The QA stance is aggressive:** any effect that doesn't cleanly map to an established encoding pattern should be flagged for manual review, not assumed correct. When in doubt, defer.
