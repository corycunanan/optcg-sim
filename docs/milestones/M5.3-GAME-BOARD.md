# M5.3 — Game Board shadcn Migration

> Migrate game board UI from hand-rolled components to shadcn primitives, improving accessibility, maintainability, and consistency with the rest of the app.

---

## Goal

Replace hand-built UI patterns across game board components with shadcn equivalents. The game board is currently the only major feature area with zero imports from `src/components/ui/`. This migration brings it in line with the admin, deck builder, social, and lobbies layers completed in M5.2.

---

- [x] M5.2 complete — shadcn component library fully migrated, all components available via barrel export
- [x] Dialog, DropdownMenu, Tooltip, HoverCard, Button, Badge all installed and token-styled

## Tasks

### Phase A: High-Priority Replacements

#### A1. Game Modals → Dialog + Button

**Files changed:**
- `src/components/game/arrange-top-cards-modal.tsx`
- `src/components/game/select-target-modal.tsx`
- `src/components/game/player-choice-modal.tsx`
- `src/components/game/optional-effect-modal.tsx`
- `src/components/game/reveal-trigger-modal.tsx`
- `src/components/game/game-board-visual.tsx` (match-ended overlay)

| Before | After |
|--------|-------|
| Hand-rolled `absolute inset-0 z-50 bg-black/60` backdrop | shadcn `Dialog` with `DialogOverlay` (`bg-overlay` token — oklch navy at 60%) |
| Custom panel `div` with inline `style={{ width }}` | shadcn `DialogContent` with game board token overrides |
| Custom header div with title + "Hide" button | shadcn `DialogHeader` + `DialogTitle` |
| Custom footer div with action buttons | shadcn `DialogFooter` |
| `ModalBtn` component (custom styled button) | shadcn `Button` with `variant` + `size="sm"` |
| Custom `<button>` elements throughout | shadcn `Button` with appropriate variants |

**Behavioral gains:** Focus trapping, ESC key dismissal (maps to "Hide"), Radix portal rendering, enter/exit animations, ARIA dialog semantics.

**Key decision:** Game modals use `Dialog` with `modal={false}` to avoid blocking the board's pointer events when modals are in "hide" state. The `open` prop is controlled: `open={!isHidden}`, `onOpenChange` maps to `onHide`.

**Code savings:** ~70 lines of manual positioning logic eliminated. `use-card-tooltip.tsx` hook deleted entirely.

## Risks

| Risk | Mitigation |
|------|------------|
| Dialog portal renders outside board scaling context | Game board is full-viewport (`fixed inset-0`), so `fixed` positioning matches `absolute inset-0` behavior |
| Dialog focus trap conflicts with drag-and-drop | Use `modal={false}` on Dialog so pointer events aren't blocked on underlying board elements |
| Tooltip conflicts with dnd-kit drag activation | Radix Tooltip dismisses on pointer down, which fires before dnd-kit's 8px activation threshold |
| DropdownMenu positioning differs from cursor-anchored context menu | Menu anchors to card element instead of cursor position — standard UX pattern, acceptable trade-off |
| `bg-overlay` token differs from `bg-black/60` | Token is `oklch(18% 0.03 245 / 0.60)` (dark navy at 60%) — visually similar, consistent with rest of app |
