# M5.4 — Game Board Motion

> Add motion.dev animations to the game board: hover/click micro-interactions and zone-to-zone card flight animations.

---

## Goal

Cards currently appear instantly when moving between zones. This milestone adds purposeful animation using motion.dev to communicate state changes and spatial relationships — hover feedback on interactive cards, tap feedback on actions, and flying card transitions when cards move between zones (hand to field, field to trash, deck to hand, etc.).

## Architecture Decision: Animation Overlay Layer

Cards get new `instanceId` values when changing zones (OPTCG rules §3-1-6), so React treats them as different elements. motion.dev's `layoutId` won't work because:
- `instanceId` is not stable across zones
- `cardId` is not unique (up to 4 copies in a deck)
- Cards render in different component trees (flex hand vs. absolute field vs. stacked life)

**Solution:** A separate overlay layer renders "ghost" cards that fly between zone positions, while real cards mount/unmount instantly underneath. The `GameState.eventLog` (append-only, with `type`, `playerIndex`, `payload.cardId`) drives transition detection — no fragile state diffing.

### Phase 2: Zone-to-Zone Movement Animations

#### 2a. Zone position registry

**New file:** `src/contexts/zone-position-context.tsx`

React context providing:
- `register(zoneKey: string, element: HTMLElement)` — called by zone components
- `unregister(zoneKey: string)` — cleanup on unmount
- `getRect(zoneKey: string): DOMRect | null` — reads viewport-space position

Zone key convention:
- Player: `"p-leader"`, `"p-char-0"` through `"p-char-4"`, `"p-stage"`, `"p-hand"`, `"p-trash"`, `"p-deck"`, `"p-life"`, `"p-don"`
- Opponent: `"o-leader"`, `"o-char-0"` through `"o-char-4"`, etc.

`getBoundingClientRect()` returns viewport coords that already account for CSS `transform: scale()`.

#### 2b. Transition detection hook

**New file:** `src/hooks/use-card-transitions.ts`

Watches `eventLog` array growth to detect card movements:
- Stores `prevEventLogLength` via `useRef`
- Maps event types to zone transitions:

| Event Type | From | To |
|-----------|------|-----|
| `CARD_PLAYED` | hand | character/stage |
| `CARD_KO` / `CARD_TRASHED` | character/stage | trash |
| `CARD_DRAWN` | deck | hand |
| `CARD_RETURNED_TO_HAND` | character | hand |
| `CARD_ADDED_TO_HAND_FROM_LIFE` | life | hand |
| `DON_PLACED_ON_FIELD` | don deck | cost area |

Returns `CardTransition[]`:
```ts
interface CardTransition {
  id: string;           // nanoid
  cardId: string;       // for rendering card image
  fromZoneKey: string;  // zone position registry key
  toZoneKey: string;
  playerIndex: 0 | 1;
  startedAt: number;
}
```

**Safeguards:**
- Max 5 concurrent animations (queue overflow)
- Auto-expire after 800ms (safety valve)
- Suppress during active drag (`activeDrag !== null`)

#### 2c. Card animation overlay

**New file:** `src/components/game/board-layout/card-animation-layer.tsx`

- Positioned `fixed inset-0 pointer-events-none` with high z-index (above board, below modals)
- For each `CardTransition`, renders `<motion.div>` wrapping a `BoardCard`:
  - `initial={{ x: fromRect.x, y: fromRect.y }}`
  - `animate={{ x: toRect.x, y: toRect.y }}`
  - `transition={cardTransitions.zoneMove}` (spring: stiffness 250, damping 28)
  - `onAnimationComplete` removes transition from active list
- Uses `AnimatePresence` for clean exit animations
- Skips overlay entirely when `prefers-reduced-motion` is active

#### 2d. Wire up in board layout

**File:** `src/components/game/board-layout/board-layout.tsx`

- Wrap content in `<ZonePositionProvider>`
- Add `useCardTransitions(eventLog)` hook
- Render `<CardAnimationLayer transitions={transitions} cardDb={cardDb} />`

#### 2e. Register zone positions

Minor ref registration additions to:
- `field-card.tsx` — character slots, leader, stage
- `hand-layer.tsx` — hand container
- `life-zone.tsx` — life stack
- `trash-zone.tsx` — trash zone
- `don-zone.tsx` — DON cost area

## Risks

| Risk | Mitigation |
|------|------------|
| `motion.div` ref conflicts with dnd-kit `useDraggable`/`useDroppable` | motion.dev forwards refs cleanly; existing `mergedRef` pattern in `PlayerFieldCard` already combines drag + drop refs — motion wraps outside |
| Board CSS scale transform breaks overlay coordinates | `getBoundingClientRect()` returns viewport coords already accounting for transforms; overlay uses `position: fixed` |
| Rapid state updates cause animation pile-up | 5-animation cap, 800ms auto-expiry, queue system, suppression during drag |
| Event log missing payload data for some events | Fallback to zone center positions from registry; `payload.cardId` confirmed present on all card movement events |
| Performance with 50+ elements | Only flying cards animate (typically 1-3 at a time); hover/tap use GPU-composited `transform` only |
