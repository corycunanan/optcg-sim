# M5.4 — Game Board Motion

> Add motion.dev animations to the game board: hover/click micro-interactions and zone-to-zone card flight animations.

---

## Goal

Cards currently appear instantly when moving between zones. This milestone adds purposeful animation using motion.dev to communicate state changes and spatial relationships — hover feedback on interactive cards, tap feedback on actions, and flying card transitions when cards move between zones (hand to field, field to trash, deck to hand, etc.).

## Architecture Decision: Animation Overlay Layer

Cards get new `instanceId` values when changing zones (OPTCG rules §3-1-6), so React treats them as different elements. motion.dev's `layoutId` won't work because:
- `instanceId` is not stable across zones
- `cardId` is not unique (up to 4 copies in a deck)
- Cards render in different component trees (flex hand vs. absolute field vs. stacked life)

**Solution:** A separate overlay layer renders "ghost" cards that fly between zone positions, while real cards mount/unmount instantly underneath. The `GameState.eventLog` (append-only, with `type`, `playerIndex`, `payload.cardId`) drives transition detection — no fragile state diffing.

### Phase 2: Zone-to-Zone Movement Animations

#### 2a. Zone position registry

**New file:** `src/contexts/zone-position-context.tsx`

React context providing:
- `register(zoneKey: string, element: HTMLElement)` — called by zone components
- `unregister(zoneKey: string)` — cleanup on unmount
- `getRect(zoneKey: string): DOMRect | null` — reads viewport-space position

Zone key convention:
- Player: `"p-leader"`, `"p-char-0"` through `"p-char-4"`, `"p-stage"`, `"p-hand"`, `"p-trash"`, `"p-deck"`, `"p-life"`, `"p-don"`
- Opponent: `"o-leader"`, `"o-char-0"` through `"o-char-4"`, etc.

`getBoundingClientRect()` returns viewport coords that already account for CSS `transform: scale()`.

#### 2b. Transition detection hook

**New file:** `src/hooks/use-card-transitions.ts`

Watches `eventLog` array growth to detect card movements:
- Stores `prevEventLogLength` via `useRef`
- Maps event types to zone transitions:

| Event Type | From | To |
|-----------|------|-----|
| `CARD_PLAYED` | hand | character/stage |
| `CARD_KO` / `CARD_TRASHED` | character/stage | trash |
| `CARD_DRAWN` | deck | hand |
| `CARD_RETURNED_TO_HAND` | character | hand |
| `CARD_ADDED_TO_HAND_FROM_LIFE` | life | hand |
| `DON_PLACED_ON_FIELD` | don deck | cost area |

Returns `CardTransition[]`:
```ts
interface CardTransition {
  id: string;           // nanoid
  cardId: string;       // for rendering card image
  fromZoneKey: string;  // zone position registry key
  toZoneKey: string;
  playerIndex: 0 | 1;
  startedAt: number;
}
```

**Safeguards:**
- Max 5 concurrent animations (queue overflow)
- Auto-expire after 800ms (safety valve)
- Suppress during active drag (`activeDrag !== null`)

#### 2c. Card animation overlay

**New file:** `src/components/game/board-layout/card-animation-layer.tsx`

- Positioned `fixed inset-0 pointer-events-none` with high z-index (above board, below modals)
- For each `CardTransition`, renders `<motion.div>` wrapping a `BoardCard`:
  - `initial={{ x: fromRect.x, y: fromRect.y }}`
  - `animate={{ x: toRect.x, y: toRect.y }}`
  - `transition={cardTransitions.zoneMove}` (spring: stiffness 250, damping 28)
  - `onAnimationComplete` removes transition from active list
- Uses `AnimatePresence` for clean exit animations
- Skips overlay entirely when `prefers-reduced-motion` is active

#### 2d. Wire up in board layout

**File:** `src/components/game/board-layout/board-layout.tsx`

- Wrap content in `<ZonePositionProvider>`
- Add `useCardTransitions(eventLog)` hook
- Render `<CardAnimationLayer transitions={transitions} cardDb={cardDb} />`

#### 2e. Register zone positions

Minor ref registration additions to:
- `field-card.tsx` — character slots, leader, stage
- `hand-layer.tsx` — hand container
- `life-zone.tsx` — life stack
- `trash-zone.tsx` — trash zone
- `don-zone.tsx` — DON cost area

### Phase 3: Balatro-Style Hand Animations

Inspired by Balatro's card dealing feel — cards fan into the hand with staggered springs, overshoot, and smooth repositioning.

#### 3a. Animated hand repositioning with `layout` transitions

**File:** `src/components/game/board-layout/hand-layer.tsx`

Convert hand cards from static flex positioning to motion.dev `layout` animations. When the hand composition changes (card added, removed, or reordered), existing cards slide smoothly to their new positions instead of snapping.

- Wrap each hand card in a `motion.div` with `layout` prop
- Use a bouncy spring (lower damping, more overshoot) for the dealing feel
- `AnimatePresence` handles cards entering/exiting the hand

#### 3b. Bouncier arrival spring for zone-to-hand flights

**File:** `src/lib/motion.ts`, `src/components/game/board-layout/card-animation-layer.tsx`

Tune the flight animation spring for cards arriving in hand to have more overshoot and settle — feels like a card being tossed rather than sliding on a rail.

- New `cardTransitions.zoneToHand` spring: lower damping (~18), moderate stiffness (~280)
- Card animation layer selects spring based on destination zone

#### 3c. Staggered multi-card draws

**File:** `src/hooks/use-card-transitions.ts`

When multiple cards are drawn in sequence (draw phase, search effects), stagger their flight animations with ~60ms delay between each card.

- Detect consecutive `CARD_DRAWN` events in the same batch
- Apply incremental `delay` to each transition's spring

#### 3d. Subtle arc trajectory on zone-to-hand flights

**File:** `src/components/game/board-layout/card-animation-layer.tsx`

Add a slight vertical arc to card flights headed to hand — cards peak upward mid-flight before settling into the hand.

- Use motion.dev keyframes for `y` axis: `[fromY, midY - arcHeight, toY]`
- Arc height proportional to flight distance (~20-30px at board scale)
- Only applied to zone-to-hand transitions, not hand-to-field

### Phase 4: State Change Animations (Rest/Activate)

Animate the ACTIVE ↔ RESTED state transition for all card types on the board. Cards rotate 90° with spring physics when resting, and spring back to 0° when activating. Opacity dims to 60% (70% for DON) when rested.

#### 4a. Motion presets for state changes

**File:** `src/lib/motion.ts`

Add `board.stateChange` section with two spring configs:
- `rest` (stiffness 300, damping 25) — decisive, snappy for attacking/blocking/cost
- `activate` (stiffness 300, damping 18) — slight bounce for refreshing

Convenience re-exports: `cardRest`, `cardActivate`

#### 4b. Strip static rotation from BoardCard

**File:** `src/components/game/board-card.tsx`

Remove static `rotate(90deg)` transform and `opacity-60` class from `isRested` logic. Rotation now animated by parent wrapper components. DON badge positioning simplified (no longer adjusts for rotation).

#### 4c. Animate field cards (leaders, characters)

**File:** `src/components/game/board-layout/field-card.tsx`

`PlayerFieldCard` and `OpponentFieldCard` — add `animate={{ rotate, opacity }}` and `transition` to existing `motion.div` wrappers. Spring selection based on destination state (rest = snappy, activate = bouncy).

#### 4d. Animate stage cards

**Files:** `src/components/game/board-layout/field-card.tsx`, `src/components/game/board-layout/board-layout.tsx`

`DroppableStageZone` — wrap `BoardCard` in `motion.div` with same animate/transition pattern. Opponent stage in `board-layout.tsx` gets matching `motion.div` wrapper.

#### 4e. Restructure DON cards for animation

**File:** `src/components/game/board-layout/don-zone.tsx`

`DonCard` — replace conditional two-DOM-tree approach (active vs. rested) with single `motion.div` that animates rotation. `DonZone` — merge split active/rested groups into single keyed list with `layout` animation for smooth repositioning when cards transition between states. Each card animated individually with per-card stagger during bulk operations.

#### 4f. Refresh phase stagger

**File:** `src/components/game/board-layout/board-layout.tsx`

Detect REFRESH → DRAW phase transition. Pass per-slot `animationDelay` to all field/stage/DON components creating a wave effect:
- Leader: 0ms, characters: 30-150ms (30ms per slot), stage: 180ms, DON: 200ms + 20ms per card

Stagger disabled when `prefers-reduced-motion` is active. Flag cleared after 500ms.

- [ ] Stagger delays suppressed when `prefers-reduced-motion` is active

## Risks

| Risk | Mitigation |
|------|------------|
| `motion.div` ref conflicts with dnd-kit `useDraggable`/`useDroppable` | motion.dev forwards refs cleanly; existing `mergedRef` pattern in `PlayerFieldCard` already combines drag + drop refs — motion wraps outside |
| Board CSS scale transform breaks overlay coordinates | `getBoundingClientRect()` returns viewport coords already accounting for transforms; overlay uses `position: fixed` |
| Rapid state updates cause animation pile-up | 5-animation cap, 800ms auto-expiry, queue system, suppression during drag |
| Event log missing payload data for some events | Fallback to zone center positions from registry; `payload.cardId` confirmed present on all card movement events |
| Performance with 50+ elements | Only flying cards animate (typically 1-3 at a time); hover/tap use GPU-composited `transform` only |
