> Install shadcn/ui and motion.dev, apply new design tokens from the branding guideline, swap the display font to DM Serif Display. The app renders in the updated type system with zero component changes.

## Tasks

### 1. Install dependencies

| Package | Purpose |
|---------|---------|
| `motion` | motion.dev — animation library (replaces need for framer-motion) |
| `shadcn` (CLI only) | `npx shadcn@latest init` — generates `components.json` config |

**Note:** shadcn/ui v2+ supports Tailwind v4 natively via `@theme inline` — the same mechanism we already use. No Tailwind config migration needed.

**shadcn init choices:**
- Style: Default
- Base color: Slate (we override everything via our tokens anyway)
- CSS variables: Yes
- `components.json` path aliases should match our existing `@/components/ui` and `@/lib`
- Do NOT let init overwrite `globals.css` — we'll manually add the shadcn aliases

### 2. Swap display font: Barlow Condensed → DM Serif Display

**File: `src/app/layout.tsx`**

Remove Barlow Condensed import. Add DM Serif Display:

```ts
// Remove:
import { Barlow_Condensed } from "next/font/google";
const barlowCondensed = Barlow_Condensed({ ... });

// Add:
import { DM_Serif_Display } from "next/font/google";
const dmSerifDisplay = DM_Serif_Display({
  subsets: ["latin"],
  weight: ["400"],
  style: ["normal", "italic"],
  variable: "--font-dm-serif-display",
  display: "swap",
});
```

Update the `<html>` className to include `dmSerifDisplay.variable` instead of `barlowCondensed.variable`.

**File: `src/app/globals.css`**

Update the Tailwind theme font mapping:

```css
@theme inline {
  --font-display: var(--font-dm-serif-display);
  /* --font-sans and --font-mono unchanged */
}
```

### 3. Add shadcn CSS variable aliases to `globals.css`

Add these inside the existing `:root` block, **below** the current tokens (additive, not replacing):

```css
/* ── shadcn/ui compatibility aliases ── */
--background: var(--surface-base);
--foreground: var(--text-primary);
--card: var(--surface-1);
--card-foreground: var(--text-primary);
--popover: var(--surface-1);
--popover-foreground: var(--text-primary);
--primary: var(--navy-900);
--primary-foreground: var(--text-inverse);
--secondary: var(--surface-2);
--secondary-foreground: var(--text-primary);
--muted: var(--surface-3);
--muted-foreground: var(--text-secondary);
--accent: var(--gold-500);
--accent-foreground: var(--navy-900);
--destructive: var(--red-600);
--destructive-foreground: var(--text-inverse);
--input: var(--border);
--ring: var(--navy-900);
```

These aliases let shadcn components reference `hsl(var(--primary))` style tokens while our actual values come from the existing oklch system. Since shadcn v2 with Tailwind v4 uses CSS variables directly (not the `hsl()` wrapper), our oklch values work as-is.

### 4. Add responsive spacing variables to `globals.css`

Add below the `:root` block:

```css
/* ── Responsive Section Spacing (Riftbound-inspired) ── */
:root {
  --section-pad-y: 64px;
  --section-pad-x: 48px;
  --content-gap: 32px;
}

@media (max-width: 1024px) {
  :root {
    --section-pad-y: 48px;
    --section-pad-x: 32px;
    --content-gap: 24px;
  }
}

@media (max-width: 600px) {
  :root {
    --section-pad-y: 32px;
    --section-pad-x: 16px;
    --content-gap: 16px;
  }
}
```

Expose in `@theme inline`:

```css
--spacing-section-y: var(--section-pad-y);
--spacing-section-x: var(--section-pad-x);
--spacing-content-gap: var(--content-gap);
```

## Risks

| Risk | Mitigation |
|------|-----------|
| shadcn init overwrites `globals.css` | Run with `--no` to decline overwrite, or back up first. We add aliases manually. |
| DM Serif Display renders differently than Barlow Condensed at existing sizes | Expected — the type scale stays the same, but character width changes. Visual review catches layout issues. Heading containers should already use flexible widths. |
| shadcn v2 + Tailwind v4 compatibility issues | shadcn v2 officially supports Tailwind v4. If issues arise, check `components.json` uses the `@theme inline` mechanism. |
| motion.dev bundle size | motion.dev tree-shakes well. We only import what we use. No impact until Phase 4/5 when we actually animate. |
