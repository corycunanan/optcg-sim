# M4.5 Next Session: Fix TypeScript Compilation Errors

## Context

We've been working through M4.5 Schema Encoding QA (see `docs/milestones/M4.5-SCHEMA-QA.md`). Phases 1-2 are complete and Phase 3 is partially done — we implemented ~440 cards worth of engine handlers across action types, cost types, target types, and duration types in a single session.

The new code was written without TypeScript compilation checks. Running `cd workers/game && npx tsc --noEmit` shows ~29 errors in our new code (ignore pre-existing test file errors in `__tests__/`).

## What Needs Fixing

Run `cd workers/game && npx tsc --noEmit 2>&1 | grep -v '__tests__'` to see only the errors in production code.

There are 5 categories of errors:

### 1. New event type literals not in `GameEventType` union
**File:** `shared/game-types.ts` — find the `GameEventType` union and add these members:
- `ATTACK_REDIRECTED` (used in `battle-actions.ts`)
- `CARD_REMOVED_FROM_LIFE` (used in `battle-actions.ts`)
- `EXTRA_TURN_GRANTED` (used in `effects.ts`)
- `EVENT_ACTIVATED` (used in `play.ts`)
- `LIFE_CARD_TURNED_FACE_UP` (used in `cost-handler.ts`)
- `LIFE_CARD_TURNED_FACE_DOWN` (used in `cost-handler.ts`)

### 2. Life card zone transfer missing CardInstance properties
**File:** `workers/game/src/engine/effect-resolver/actions/battle-actions.ts`
In `executeDealDamage` and `executeSelfTakeDamage`, life cards moved to hand via `{ ...lifeCard, zone: "HAND" as const }` are missing required `CardInstance` properties (`state`, `attachedDon`, `turnPlayed`, `controller`, `owner`). Look at how other handlers (e.g., `LIFE_TO_HAND` in `life.ts`) construct the full CardInstance when moving cards to hand.

### 3. Filter type defaults to `{}`
**File:** `workers/game/src/engine/effect-resolver/actions/draw-search.ts`
In `executeSearchTrashTheRest`, the line `const filter = p_.filter ?? {};` types `filter` as `{}`, so property accesses like `filter.traits` fail. Cast it: `const filter = (p_.filter ?? {}) as Record<string, any>;`

### 4. `extraTurnsPending` not on `TurnState`
**File:** `shared/game-types.ts` — find the `TurnState` interface and add:
```typescript
extraTurnsPending?: number;
```
This is used by `executeExtraTurn` in `effects.ts`.

### 5. `restDestination` not in `PromptOptions`
**File:** `shared/game-types.ts` — find the `PromptOptions` interface/type and add `restDestination?: string`. This is used by `executeSearchTrashTheRest` in `draw-search.ts` to pass the rest destination through the prompt.
